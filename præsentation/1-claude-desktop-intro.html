Chat kommer til kort når proces

  • 1
    Brugeren bliver procesmotorChatten kan lave output, men brugeren skal selv drive filvalg, næste trin og overlevering.
  • 2
    Kvalitetssikring kræver eksplicit promptReview, krydstjek og forbedring sker kun sikkert, hvis brugeren beder om det og følger op.
  • 3
    Leverancen lever i sessionenRapport, pptx og artifact kan laves, men arbejdsprocessen er ikke i sig selv et lokalt projekt med filstyring.
  • 4
    Gentagelse bliver manuelNæste måned starter man ofte forfra med data, kontekst, prompt og kontrolpunkter.
Risiko ved Chat connectors

Chat er størst risikomæssigt, når den får connectors til confidential data. Så er det ikke længere bare en samtale, men en samtale med adgang til interne kilder.

WCowork projekt

Cowork hjælper når workflow

  • 1
    Opgaven har filerClaude kan arbejde i en connected folder frem for kun i samtalen.
  • 2
    Output skal reviewesUdkast, rettelser, review-noter og status kan samles i samme flow.
  • 3
    Kontekst skal leve videreProjektfiler, instruktioner og memory holder opgaven samlet.
  • 4
    Opgaven gentagesScheduled tasks kan køre, når computeren er vågen og Desktop er åben.
Videre til Code når system

Beregninger skal industrialiseres, der skal bygges scripts, SQL-checks, tests, servere, pipelines eller egentlige integrationer.

Risiko ved Cowork arbejdsrum

Shell/kode kører isoleret i VM, ikke frit på hele maskinen. Risikoen ligger især i rigtige filer, connectors, computer use, web access og skrivning tilbage til arbejdsmapper.

#Claude Code workspace

Code hjælper når udvikling

  • 1
    Processen skal gentagesRapporten bygges fra scripts, data og tests i stedet for manuelt arbejde.
  • 2
    Der er repo og kodeClaude kan læse, ændre og teste på tværs af filer.
  • 3
    Servere skal køreBruges til Python, Node, dev-servere, CLI'er og testværktøjer.
  • 4
    Review skal være tekniskDiff, lint, tests og CI-fejl kan bruges som feedbackloop.
Code kommer til kort når governance

Opgaven primært handler om forretningstekst, fortolkning eller beslutninger. Code kræver også repo, miljø, testdata og klare permissions.

Risiko ved Code blast radius

Risikoen stiger med shell, filsystem, netværk, web fetch, interne data og skip permissions. Brug sandbox, allowlists og testdata.

Projects kontekst

Et projekt samler relevante chats, instruktioner og projektviden, så Claude forstår baggrund, dokumenter og mål for et afgrænset arbejde.

Bruges tilAfdelingscases, politikker, rapportpakker, gentagne opgaver.
GovernanceHvem må se projektet, og hvilke filer hører til?
Skills arbejdsmåde

Skills er mapper med instruktioner, scripts og ressourcer, som Claude kan indlæse dynamisk for at udføre specialiserede opgaver mere ensartet.

Bruges tilBrandguides, analysemetoder, dokumentproduktion, faste tjeklister.
GovernanceGodkendte skills kan standardisere kvalitet og compliance.
Connectors adgang

Connectors forbinder Claude til værktøjer, datakilder og services. De kan give Claude mulighed for at søge, læse eller handle på vegne af brugeren.

Bruges tilM365, GitHub, interne systemer, databaser og MCP-services.
GovernanceLæs/skriv-rettigheder, audit, scope og tool access per samtale.
